# Security Policy

## Supported Versions

The following versions of **INFRA DRISHTI** currently receive security updates and support.

| Version | Supported          |
| ------- | ------------------ |
| Latest Release | ✅ |
| Main (Development Branch) | ✅ |
| Older Versions | ❌ |

We strongly recommend using the latest version to ensure maximum security and stability.

---

## Reporting a Vulnerability

The INFRA DRISHTI team takes security vulnerabilities seriously.  
If you discover a vulnerability, please report it responsibly.

### 🚨 How to Report

Please **DO NOT** open a public GitHub issue for security vulnerabilities.

Instead, report the issue privately via:

📧 **Email:** infra.drishti.project@gmail.com  
*(Replace with your official project email if needed)*

---

### 📋 Please Include

When reporting a vulnerability, include:

- Description of the issue
- Steps to reproduce the vulnerability
- Possible impact
- Screenshots or proof of concept (if available)
- Suggested fix (optional)

---

### ⏱️ Response Timeline

After submission:

- ✅ Acknowledgement within **48 hours**
- 🔍 Investigation and validation of the report
- 🛠️ Fix development if confirmed
- 📢 Security update released when necessary

You may be credited for responsible disclosure if you wish.

---

### 🤝 Responsible Disclosure

We request that security researchers:

- Avoid accessing or modifying real user data
- Do not exploit vulnerabilities beyond proof-of-concept
- Do not publicly disclose issues before a fix is released
- Act in good faith to protect users and the project

---

### 🔐 Our Commitment

INFRA DRISHTI is committed to maintaining a secure platform focused on public infrastructure safety.  
We appreciate responsible security research that helps improve the reliability and trustworthiness of our project.
